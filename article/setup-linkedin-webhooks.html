Setup Linkedin Webhooks

Created At：2023-08-21 10:49 Updated At：2023-09-04 18:00

To get alerts via LinkedIn's API webhooks whenever your company makes a post, you will need to follow these general steps:

  1. Register your app on the LinkedIn Developer Platform:

  • Go to the LinkedIn Developer Platform website and sign in with your LinkedIn account.

  • Create a new app and provide the necessary information such as the app name, description, and website URL.

  • Select the appropriate API products for your app, in this case, the Marketing Developer Platform.

  • Set up your app's authentication and authorization by creating a LinkedIn app access token.

  1. Subscribe to company page updates:

  • Use the LinkedIn Marketing Developer Platform API to subscribe to updates for your company page.

  • You will need to provide your LinkedIn app access token and the appropriate API endpoints for subscribing to updates.

  1. Set up webhooks to receive alerts:

  • Use the LinkedIn Marketing Developer Platform API to set up webhooks for receiving alerts.

  • You will need to provide the appropriate API endpoints for receiving the webhook notifications.

  • Make sure that your webhook endpoint is secure and can handle the expected traffic.

  1. Test your app and webhook:

  • Create a test post on your company page to ensure that your app is receiving the webhook notifications.

  • Make sure that the webhook payload includes the information you need, such as the post text and image.

  1. Handle errors and manage alerts:

  • Handle any errors that may occur, such as failed webhook deliveries or invalid payloads.

  • Manage the alerts you receive by storing them in a database or sending them to a notification service.

Please note that the exact steps may vary depending on your specific use case and the programming language and frameworks you are using. You may also need to refer to LinkedIn's developer documentation for more detailed information on using their API and webhooks.
